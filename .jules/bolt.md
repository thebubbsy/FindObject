## 2026-04-12 - PowerShell Pipeline Performance Idioms
**Learning:** PowerShell pipeline `process` blocks execute once per item and are extremely sensitive to microscopic overhead. String operations (like concatenating wildcards `"*$keyword*"`) or typecasting (`.ToString()`) inside the block create severe bottlenecks. The `foreach` loop enumerator also introduces significant overhead compared to a standard `for` loop with a pre-cached array length. Furthermore, `return` in a `process` block correctly behaves like a `continue` statement, allowing for early exits without breaking the pipeline.
**Action:** When optimizing PowerShell pipeline functions, always pre-calculate data (like wildcards) in the `begin` block. Use `$item -isnot [type]` for fast type checking. Replace `foreach` loops with standard `for ($i=0; $i -lt $count; $i++)` loops when iterating over fixed arrays. Utilize early `return` to terminate redundant condition checking immediately.
