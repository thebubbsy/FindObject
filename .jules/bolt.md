## 2026-04-02 - Optimize PowerShell Pipeline Process Block Throughput
**Learning:** In PowerShell, the pipeline `process` block is highly sensitive to throughput. Code execution here acts as a multiplier per object. String interpolation (e.g. `"*$keyword*"`) should not occur within the `process` block. It must be pre-calculated in the `begin` block and scoped using standard variables, avoiding `$script:` scope. Redundant loop constructs `foreach` are slower than standard `for` loops caching the arrays length (`$c = $arr.Length`). Finally, reflection operations (such as `.GetType().FullName`) within high-volume string formats like `Write-Verbose` are evaluated even when verbose output is disabled, resulting in massive performance penalties. Calling `.ToString()` on arbitrary inputs can throw errors, and instead, casting `-isnot [string]` followed by `[string]$var` is faster and safer.
**Action:** When filtering objects via pipeline, always pre-compute variables and formats in the `begin` block. Within the `process` block, substitute `foreach` with bounded `for` loops, use type-casting over method calls for type conversion, and rigorously remove any object reflection or complex interpolation from conditional or logging statements (even seemingly inert `Write-Verbose` calls). Always prefer early `return` to emit objects and instantly break execution flows.
