## 2025-04-08 - Optimize High-Throughput PowerShell Pipeline Blocks
**Learning:** In PowerShell pipeline advanced functions, arguments passed to logging commands like `Write-Verbose` are evaluated *before* the command executes, even if verbose logging is disabled globally. Doing reflection like `$InputObject.GetType().FullName` or performing string interpolation (`"*$keyword*"`) inside high-throughput `process` blocks creates massive hidden performance penalties due to redundant per-object allocations and evaluations.
**Action:** Always extract static calculations (like wildcard string interpolation) to the `begin` block to encapsulate them in function-scoped variables. Substitute reflection-based error logging with simpler operations, swap `foreach` loops for standard `for` loops with pre-cached `.Length`, and utilize early `return` to avoid maintaining tracking variables (like `$match`) inside `process` blocks.
