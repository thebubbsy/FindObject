## 2024-05-24 - Optimizing the PowerShell Pipeline
**Learning:** PowerShell pipeline performance is heavily bottlenecked by string interpolation (like `"*$keyword*"`) inside `process` blocks. Also, evaluating object types dynamically within strings (e.g., `Write-Verbose "$($InputObject.GetType().FullName)"`) inside `process` blocks carries a hidden performance penalty, even when Verbose is off. Finally, early returns inside `process` block are highly effective to skip objects cleanly without terminating the pipeline.
**Action:** Pre-calculate patterns in the `begin` block and store them in function-scoped variables. Avoid dynamic evaluation strings in logging output unless needed, and use early `return` to jump to the next pipeline object instead of carrying state variables to the end of the block.
