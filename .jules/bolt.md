## 2024-05-18 - PowerShell Pipeline String Interpolation Bottleneck
**Learning:** String interpolation (`"*$keyword*"`) performed inside the `process` block for every single object processed by the pipeline introduces a massive performance bottleneck. The overhead of repeatedly allocating and building strings is significant in high-throughput operations. Additionally, calling `.ToString()` unconditionally on pipeline objects is slower than performing a type-check (`-isnot [string]`) followed by a direct cast. Finally, using a standard `for` loop with a pre-cached array `.Length` is noticeably faster than using `foreach` inside the `process` block.

**Action:** Pre-calculate any wildcard patterns or strings that only depend on initial arguments in the `begin` block and store them in function-scoped variables (e.g., arrays). Replace `.ToString()` calls with type casting where applicable, and utilize standard `for` loops combined with early `return` to skip processing the remaining patterns or objects.
