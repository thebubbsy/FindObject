## 2026-04-04 - Optimize pipeline string interpolation logic
**Learning:** In PowerShell advanced functions, putting string interpolation directly inside the `process` block (like `"*$keyword*"`) or executing reflection for debugging strings (like `$($InputObject.GetType().FullName)`) forces evaluation for every object in the pipeline, which creates a massive performance penalty during high-throughput processing. Also, `$match = $true`/`$false` state variables can be optimized away using early `return` to immediately advance pipeline execution.
**Action:** Pre-calculate interpolated strings in the `begin` block to create simple arrays of raw strings, replace `foreach` with `for` loops caching the string count, strip heavy reflection from `Write-Verbose` within the `process` block, and use early `return` inside loops.
